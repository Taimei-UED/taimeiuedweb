# LUMA 方法论概述

LUMA 是一套系统化的设计思维方法论，由 LUMA Institute 提出。它将创新过程拆解为三个核心方法群：**Looking**、**Understanding** 和 **Making**。

## 核心理念

LUMA 的核心理念是：**创新不是天赋，而是可以通过方法习得的技能**。

  3. **Making** — 原型和测试，将想法变为可验证的方案

## 方法组合（Recipes）

LUMA 强调方法不是孤立使用的，而是像食谱一样组合搭配。一个典型的创新流程：

> **Looking**（观察） → **Understanding**（理解） → **Making**（创造）

但顺序并非固定 — 你可以在任何阶段灵活组合方法。

## 适用场景

| 场景 | 推荐方法组合 |
|------|-------------|
| 用户研究 | Interview + Observation + Persona |
| 问题定义 | Stakeholder Mapping + Journey Mapping |
| 创意发散 | Brainstorming + Concept Poster |
| 方案验证 | Prototyping + Usability Testing |
| 服务设计 | Service Blueprint + Experience Mapping |

## 团队实践原则

1. **以人为本** — 所有方法的核心是理解人
2. **迭代循环** — 不断在三个方法群之间循环
3. **协作参与** — 跨职能团队共同参与
4. **可视思考** — 用画布、图表、原型来思考和沟通
5. **原型驱动** — 尽早制作原型，快速验证假设
