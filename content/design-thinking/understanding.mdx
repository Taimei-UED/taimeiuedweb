# Understanding：理解方法

Understanding 是将观察阶段收集的碎片信息进行系统化分析和综合，从中发现洞察和机会。

## 从数据到洞察

原始观察数据本身没有价值，价值来自于分析和综合：

> 数据 → 信息 → 知识 → 洞察 → 机会

## 核心方法

### 1. Persona（用户画像）

将观察到的用户行为模式提炼为典型角色：

<Card>
  <CardHeader>
    <CardTitle>用户画像要素</CardTitle>
    <CardDescription>一个好的画像是鲜活的、可决策的</CardDescription>
  </CardHeader>
  <CardContent>

  - **基本信息** — 姓名、年龄、职业
  - **目标** — 想完成什么？
  - **行为** — 如何使用产品？
  - **痛点** — 遇到什么困难？
  - **引言** — 一句代表TA的话

  </CardContent>
</Card>

> 画像不是统计报告，而是决策工具。不要超过 3-5 个画像。

### 2. Journey Mapping（用户旅程地图）

可视化用户从开始到完成目标的完整体验：

```
阶段:  认知 → 评估 → 使用 → 推荐
行为:  搜索 → 对比 → 注册 → 分享
情绪:  😐     😕     😊     😄
痛点:  找不到  太复杂  顺利   缺激励
机会:  SEO   简化    保持   激励计划
```

旅程地图的关键价值在于发现**情绪低谷** — 那里就是设计的机会。

### 3. Stakeholder Mapping（利益相关者图谱）

识别所有与产品相关的人和组织，理解他们的关系和影响力：

1. **核心用户** — 直接使用产品的人
2. **影响者** — 影响决策的人
3. **决策者** — 决定是否购买/使用的人
4. **支持者** — 提供支持的人（客服、运维）

### 4. Affinity Clustering（亲和聚类）

将大量观察记录按相似性分组，发现隐藏模式：

- 把每条观察写在便签上
- 在白板上按相似性自然聚类
- 为每个聚类命名
- 识别最大的聚类 — 那是核心问题

## 综合输出

Understanding 阶段的最终产出应该回答：

| 问题 | 产出 |
|------|------|
| 用户是谁？ | 用户画像 |
| 用户经历了什么？ | 旅程地图 |
| 核心问题是什么？ | 问题陈述 |
| 机会在哪里？ | 机会列表 |

## 实践建议

- **协作分析** — 不要独自分析，团队一起做
- **可视化** — 把分析结果贴在墙上
- **保持诚实** — 不要为了结论而扭曲数据
- **可行动** — 每个洞察都应指向行动
