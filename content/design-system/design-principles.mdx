# 设计原则

设计原则是团队在设计过程中遵循的基本准则，它们为每一个设计决策提供方向和判断依据。

## 一致性

> 一致性是信任的基础。

在视觉语言、交互模式和信息架构上保持高度一致。用户在产品的任何角落都应感受到统一的体验。

- **视觉一致性**：颜色、字体、间距遵循 Design Tokens
- **交互一致性**：相似操作使用相同的交互模式
- **语言一致性**：文案风格、术语使用统一

## 层次与优先级

通过视觉层次引导用户注意力，突出最重要信息：

1. **主要操作**应最显眼 — 使用 <Button variant="primary">主按钮</Button>
2. **次要操作**应弱化 — 使用 <Button variant="secondary">次按钮</Button>
3. **辅助信息**应低调 — 使用灰色或更小字号

## 可访问性

设计应面向所有用户，包括使用辅助技术的人群：

| 原则 | 要求 | 实践 |
|------|------|------|
| 对比度 | WCAG AA 标准 (4.5:1) | 使用 Design Tokens 中的色阶 |
| 焦点可见 | 键盘导航可见焦点 | `focus-visible:ring-2` |
| 语义化 | 使用语义化 HTML | `<button>` 而非 `<div>` |
| 文字大小 | 最小 14px | `text-sm` 起步 |

## 留白

留白不是浪费，而是呼吸的空间。合理的留白能：

- 提升信息可读性
- 建立视觉层次
- 减少认知负担
- 营造高级感

---

设计原则不是教条，而是在面对取舍时的判断框架。当原则之间冲突时，以用户价值为最高优先级。
